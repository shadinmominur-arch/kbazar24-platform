Owner directive (2026-06-10): keep GEO and AEO in mind for every SEO/content task going forward.

- **GEO** — optimize so AI Overviews/ChatGPT/Perplexity/Gemini cite Emart content: complete `Product`/`FAQPage`/`Article`/`Organization` schema, definition-first sentences, factual specificity (sizes/ingredients/prices/origin), clean labeled content sections.
- **AEO** — optimize for featured snippets/"People Also Ask"/voice: concise direct-answer paragraphs, FAQPage schema (PDPs already have 5 Qs/product), short How-to/step lists.

**Why:** most Bangladesh traffic is mobile via Android's default Google search, which increasingly surfaces AI Overviews/Gemini answers ahead of organic links — and TikTok/Facebook/YouTube trending K-beauty topics shape what users search next.

**How to apply:**
- Full standing-consideration text lives in `workspace/SEO_MASTER.md` under "🧭 STANDING CONSIDERATION — GEO & AEO" — read it whenever doing SEO work (per project CLAUDE.md, SEO work already requires reading `SEO_MASTER.md` first).
- Any new content type (blog posts, future `/best/*`, `/compare/*`, ingredient/concern pages) should ship with appropriate schema (`Article`/`FAQPage`/`HowTo`) from day one, not retrofitted later.
- Blog topic selection (see [[project_meta_blog_resume_20260610]]) should weigh GSC data + TikTok/FB/YouTube trends, not just evergreen keywords.
- Don't block Google-Extended/AI crawlers in `robots.ts` without an explicit owner decision — that would remove GEO citation opportunities.
- Linked: still owe the fuller strategic synthesis (C3 in TASKS.md) on Android search dominance + AI search surfaces + social trending topics.
